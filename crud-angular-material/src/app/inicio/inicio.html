Angular Estudo sobre Angular

1. Introdução

Neste estudo vamos focar a criação de uma tela de cadastro de cliente, usando Angular + Angular Material, dentro do projeto que você já tem: crud-angular-material. A ideia é praticar os componentes de interface, formulários reativos, validações, integração com serviço/fake backend e utilização dos componentes visuais do Angular Material.

2. Objetivos da tela

  • Permitir ao usuário incluir um cliente com dados como nome, e-mail, cpf e data de nascimento.
  • Permitir ao usuário editar os dados de um cliente existente.
  • Validar os campos do formulário (por exemplo: e-mail válido, cpf obrigatório, data mínima).
  • Exibir feedback visual ao usuário (via componentes Angular Material como MatSnackBar, MatDialog, MatFormField, MatInput etc.).
  • Estilização e layout responsivo usando Angular Material e possivelmente @angular/flex-layout.
  • Integrar com um serviço (pode ser simulado) para gravar/ler dados de cliente.

3. Estrutura e organização do projeto

No repositório crud-angular-material, recomenda-se:

  • Criar um módulo específico para a funcionalidade de cliente, por exemplo cliente.module.ts, que declare componentes como cliente-cadastro.component.ts e cliente-lista.component.ts.
  • No módulo principal (app.module.ts) importar o módulo de cliente.
  • Criar também um módulo para o Material: material.module.ts, que exporta todos os módulos do Angular Material que serão utilizados.
  • Criar um serviço cliente.service.ts que forneça métodos como listar(), obterPorId(), salvar() e excluir().

4. Layout da tela de cadastro

  • Uma MatToolbar no topo com o título “Cadastro de Cliente”.
  • Um formulário dentro de um cartão (MatCard):
    • Campo Nome → MatFormField + MatInput
    • Campo E-mail → MatFormField + MatInput (tipo e-mail)
    • Campo cpf → MatFormField + MatInput
    • Campo Data de Nascimento → MatFormField + MatDatepicker
  • Botões no final do formulário: “Salvar” (botão primário) e “Cancelar” (botão simples).
  • Feedback: ao salvar com sucesso, utilizar MatSnackBar com a mensagem “Cliente salvo com sucesso”.
  • Em modo de edição: pré-popular os campos com dados existentes.

5. Validações do formulário

  • nome: obrigatório, mínimo de 3 caracteres.
  • email: obrigatório, padrão e-mail válido.
  • cpf: obrigatório.
  • dataNascimento: obrigatório, e deve ser uma data anterior à data atual.
